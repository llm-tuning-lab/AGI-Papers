# flash-moe: 48GB 맥북에서 Qwen3.5-397B-A17B 돌리기

1. 개발 목적: RAM 용량이 턱없이 부족한 노트북(예: 48GB 맥북)에서 400B(4천억 파라미터) 크기의 초거대 MoE 모델을 구동하기 위해 만든 커스텀 추론 엔진입니다.

2. SSD 스트리밍: 모델 가중치를 전부 램에 올리지 않습니다. 필수 공통 레이어만 램에 상주시키고, 매 토큰을 계산할 때마다 수많은 전문가(Expert) 중 선택받은 소수의 가중치만 초고속 NVMe SSD에서 램으로 순간적으로 퍼 올린 뒤 연산합니다.

3. 애플 실리콘 버프: 맥북 M시리즈 특유의 '통합 메모리(Unified Memory)' 구조와 애플의 'Metal GPU API'를 극한으로 쥐어짜서 메모리 대역폭 병목을 최소화했습니다.

4. 로우레벨 C/C++ 구현 및 OS 신뢰 (Trust the OS): 파이썬 생태계를 최대한 거치지 않고 C언어(Objective-C)로 작성되었습니다. 특히 복잡한 메모리 맵핑(mmap)이나 커스텀 캐시를 썼다가 오히려 속도가 하락하는 것을 확인하고, 가장 원시적인 pread() 함수와 '운영체제(OS)의 기본 페이지 캐시'에 모든 I/O를 전적으로 맡겨 병목을 뚫어냈습니다.

한 줄 결론: 맥북 하드웨어와 MoE 모델의 특성을 완벽히 이해하고, C언어로 쓸데없는 최적화 꼼수(mmap 등)를 버린 뒤 OS의 순정 기능을 극한으로 끌어올린 SSD 오프로딩의 끝판왕입니다. 

(그래서 윈도우/일반 시중 PC에서는 똑같이 따라 하기가 물리적으로 힘듭니다. 저는 실제로 안 써봐서 실제로도 잘 되는지는 모르겠습니다.)

# HYPERAGENTS: 스스로 '공부법'을 코딩하며 진화하는 AI

스스로 진화하는 AI를 많이 들어보셨을 겁니다.

하지만 지금까지의 대부분의 자가 발전은 인간이 정해준 규칙과 갇힌 트랙 안에서 빙글빙글 도는 수준이었습니다. 

HYPERAGENTS는 LLM이 단순히 정답을 찾는 것을 넘어, '공부하는 방법' 자체를 스스로 파이썬 코드를 짜면서 배우는 방식을 보여줍니다.

이 논문이 왜 흥미로운지, 그리고 동시에 현실적 한계까지 4가지로 정리합니다.

1. 고정된 한계를 부수다: 두 개의 자아 (Hyperagents)

기존 AI는 인간이 짜놓은 프롬프트나 고정된 방식(메타 메커니즘)으로만 스스로를 개선했습니다. 

하지만 이 논문은 목표를 수행하는 '테스크 에이전트'와 이 에이전트의 코드를 수정하는 '메타 에이전트'를 하나의 프로그램으로 합쳤습니다. 

- 비유하자면: 단순히 수학 문제를 100개 푸는 학생이 아니라, 100개 문제를 풀때마다 "내가 왜 자꾸 틀리지? 오답 노트를 만들고, 시간 분배 전략을 새로 짜야겠다"라며 자신만의 공부법을 발명해 내는 학생을 만든 셈입니다.

2. 4가지 영역에서의 파괴적인 성과, 그리고 현실적인 숫자

연구진은 로봇 제어(보상 설계), 논문 리뷰, 코딩, 수학 채점이라는 4가지 전혀 다른 분야에 이 AI를 던져놓았습니다.

- 코딩 (버그 수정): 8.4% -> 26.7%
- 논문 심사 (채택/거절): 0.0% -> 71.0%
- 로봇 점프력 상승: 0.060 -> 0.372
- 올림피아드 수학 채점: 56.1% -> 60.1%

여기서 현실적인 질문 하나. "코딩 성공률 26.7%? 100개 중 73개는 못 고친다는 건데, 이게 쓸모가 있나요?"

맞습니다. 당장 내일 실무 프로젝트에 "네가 알아서 개발해"라고 던져둘 수준은 절대 아닙니다. 

하지만 이 수치가 흥미로운 이유는 '결과'가 아니라 '과정' 때문입니다. 인간이 개입하지 않았는데도 AI가 스스로 '과거의 실패를 기록하는 영구 메모리'와 '오류 편향 분석기'를 코딩해 내며 8점에서 26점까지 '스스로 성장했다'는 그 궤도 자체가 재미있는 포인트입니다.

3. 100번의 요행? 사기가 아닌 순수 1회차 실력 (Pass@1)

AI가 100번을 반복해서 진화했다고 하니, "100개 던져서 1개 맞춘 거(Pass@100) 아니냐?"라고 의심할 수 있습니다. 요즘 논문들이 흔히 쓰는 꼼수니까요.

- 이 모델이 말하는 100번의 반복은 시험장에서의 꼼수가 아니라, 시험 전 '훈련 단계'에서 스스로 코드를 100번 깎고 다듬으며 진화한 횟수입니다. 

- 그렇게 완성된 최종 에이전트는 실전 테스트에서 단 한 번의 기회(Pass@1)만 부여받았고, 피드백 없이 그 자리에서 26.7%를 통과해 냈습니다. 철저하게 실력으로 증명한 수치입니다.

4. 명확한 한계: 갇힌 놀이터 안의 진화

그렇다고 이 논문이 만능은 아닙니다. 연구진도 인정하는 치명적인 한계가 존재합니다.

- 뇌(가중치)는 그대로: 이 시스템은 AI의 프롬프트나 파이썬 코드 같은 '도구' 시만을 뜯어고칠 뿐, 기반이 되는 LLM 자체의 가중치를 학습시키지는 못합니다. 공부 방법론은 기가 막히게 바꿨지만, 타고난 IQ 자체를 업데이트한 건 아니라는 뜻입니다.

- 가성비 최악: 이 에이전트 하나를 진화시키는 데 들어간 API 비용만 무려 8,860만 토큰입니다. 컴퓨팅 자원 소모가 극심하여 당장의 산업적 실용성은 거의 제로에 가깝습니다.


마치며: 실용성은 없지만, 미래를 보여주다

실용성을 따지는 현업의 눈높이에서 이 AI는 가성비 최악의 장난감으로 보일 수 있습니다. 하지만 "LLM이 인간의 통제를 벗어나, 자신의 코드를 직접 수정하며 학습법 자체를 개조하기 시작했다"는 가능성(Proof of Concept)을 이만큼 증명한 연구는 드뭅니다.

단순히 정해진 문제를 푸는 도구에서, 자신을 스스로 진화시키는 개방형 지능으로 넘어가는 거대한 패러다임의 전환. HYPERAGENTS는 LLM의 다음 목적지를 보여주고 있습니다. 

어쩌면 이는 RLVR 과정과 합쳐지면서 실용성을 가질지도 모르겠습니다.

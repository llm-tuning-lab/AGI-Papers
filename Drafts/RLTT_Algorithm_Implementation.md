생각의 궤적에 보상을 주다: RLTT 알고리즘 구현의 핵심

LLM 추론 능력을 극대화하는 'Looped Language Models (LoopLM)'. 자기 스스로 여러 번의 내부 루프(Refinement steps)를 돌며 정답을 다듬는 이 강력한 기술에도 고질병이 있습니다.
바로 기존의 GRPO 같은 강화학습은 "가장 마지막 출력 텍스트"에만 보상(Reward)을 준다는 점입니다. 이렇게 되면 "중간에 어떤 고민을 하며 사고 교정을 했는지"에 대한 학습 신호가 모델 내부에 제대로 전달되지 않는 치명적 병목이 발생합니다.

이 문제를 부수는 알고리즘이 바로 **RLTT (Reward Latent Thought Trajectories)**입니다. 최종 결론뿐 아니라 생각의 모든 중간 궤적에 직접적인 보상을 꽂아주는 이 알고리즘을 C++이나 파이토치 단에서 어떻게 코드 레벨로 구현(Implementation)하는지 4가지 스텝으로 구조화해 드립니다.

1. 핵심 아이디어: "모든 루프의 Log-prob을 추출하라"
일반적인 모델은 마지막에 한 번의 확률값(Logit)을 내뱉지만, RLTT를 구현하려면 LoopLM이 내부 루프 t=1부터 최종 루프 T_max까지 도는 동안 발생하는 모든 중간 토큰의 예측 로그 확률값을 전부 추출해서 메모리에 기록해야 합니다. 이것이 이 논문의 절대적인 뼈대입니다.

2. 어드밴티지(Advantage) 계산: (Rollout 및 그룹화)
생성(Rollout)을 진행합니다. GRPO처럼 동일한 프롬프트로 여러 개의 답변 궤적을 뽑아냅니다.
- 궤적 결과가 정답과 맞으면 보상 +1, 틀리면 0을 줍니다. 
- 그리고 이 8개의 보상 값들로 평균과 표준편차를 구해 '그룹 정규화된 어드밴티지' 수치를 각 궤적에 부여합니다.

3. 핵심 수식 구현: 루프 가중합(Weighted Sum) 역전파
기존 PG Loss는 마지막 확률값과 어드밴티지만 곱했습니다. RLTT 커스텀 Loss 구현의 핵심은 '가중합'입니다.
- 각 루프 단계마다 사용자가 정해둔 **가중치(ω)**를 그 단계의 예측 확률에 곱한 후, 이 값들을 모두 더해줍니다. 
- 가중치는 그냥 모든 루프에 똑같이 분할(Uniform)하거나, 후반부 루프에 가중치를 더 주는 점진적(Progressive) 방식을 코드에 선언하면 됩니다.
- 여기에 어드밴티지를 곱하고, 망각 방지를 위한 참조 모델(Reference Model)과의 KL 페널티를 빼서 Loss를 구한 뒤 역전파(Backprop) 파이프라인을 태웁니다.

4. 엔지니어링 팁: VRAM (메모리 OOM) 사수하기
알고리즘 연산량 자체는 가중합뿐이라 가볍지만, 메모리가 터질(OOM) 확률이 압도적으로 높습니다. 
중간 루프의 모든 궤적(Log-probs)을 메모리에 수직으로 들고 있어야 하기 때문입니다. 따라서 vLLM이나 로컬 훈련 스크립트(PPO_max_token_len 등) 세팅에서 GPU당 할당되는 배치 크기나 맥스 토큰 리미트를 평소 GRPO 세팅의 최소 절반 이하(예: 8,192 수준)로 빡빡하게 줄여놓고 실험을 시작하는 것이 필수입니다. 

💡 마치며: LLM 자율 추론 통제의 완결판
단순히 "생각을 길게 해라"라고 명령(Prompt)하는 시기를 넘어섰습니다. RLTT는 모델이 "어떤 방향으로 길고 고통스럽게 고민해야 정답에 최단 거리로 수렴하는가"에 대한 내비게이션 좌표를 가중치 깊숙이 박아 넣는 핵심 트레이닝 기법입니다.

LLM 파괴적 망각 방지법: 머리가 나빠지지 않고 새 지식만 배우는 희소 메모리 파인튜닝

AI 모델을 기업 서비스에 배포하고 매일 새로운 사내 뉴스나 고객 맞춤형 지식을 가르치다 보면 큰일이 생깁니다. 새로운 걸 배우려다 기존에 잘 알던 지식 생태계마저 붕괴되어 모델이 멍청해지는 현상, 바로 '파괴적 망각(Catastrophic Forgetting)'입니다.

최근 Ludovico Bessi가 소개한 '희소 메모리 파인튜닝(Continual Learning via Sparse Memory Finetuning)' 기술은 이 엔터프라이즈 AI의 가장 뼈아픈 고질병을 수술 없이 도려냅니다.

뇌 전체를 뒤흔드는 기존 방식 대신, 새로운 사실을 저장할 '빈 방'만 찾아 지식을 밀어 넣는 우아한 공식을 4가지로 소개합니다.

1. 문제의 원인: 파라미터 간섭 (Parameter Interference)
기존 방식인 전체 파인튜닝(Full Finetuning)이나 일반 LoRA로 새로운 지식을 주입하면, 기존의 일반적인 언어 능력에 최적화되어 있던 파라미터(뇌세포) 값들이 통째로 흔들립니다. 새로운 정보를 구겨 넣으려다 과거 지식의 균형이 무너지는 역효과가 납니다.

2. 해결책 기본: FFN을 '메모리 레이어'로 교체
기존 트랜스포머 아키텍처의 FFN(Feed-Forward Network) 계층을 수많은 슬롯(Slot)들로 이루어진 거대한 '키-값 풀(Key-Value pools)'로 바꿉니다. 
이제 전체 파라미터를 멍청하게 업데이트하는 대신, 매번 튜닝할 때마다 이 슬롯 중 극소수(Tiny fraction)의 특정 슬롯만 타겟팅해 값을 넣습니다.

3. 솎아내는 영리함: TF-IDF로 방을 나눈다
그럼 어떤 방을 잠그고 어떤 방에 새 지식을 넣을까요? 정보 검색에서 흔히 쓰이는 TF-IDF 랭킹 기법을 씁니다.
- 기존 지식 절대 방어: 기존 사전 학습 시 획득한 '일반적인 언어 능력이나 보편적 지식'을 담당하는 슬롯을 찾아내어 아예 마스킹(잠금 처리) 해버립니다. (절대 건드리지 않음)
- 새로운 팩트 주입: TF-IDF를 돌려 오직 '새로운 데이터 특성'을 처리하는 데만 반응하는 특정 슬롯을 찾아내어 그 부분만 집중적으로 튜닝합니다.

4. 압도적인 결과: 망각을 11%로 봉쇄하다
이 방식을 질의응답(QA)에 적용하자 입이 떡 벌어지는 결과가 나왔습니다.
기존 전체 파인튜닝과 지식 학습 속도는 동일하게 유지하면서도, 이미 알고 있던 기존 지식을 까먹었는지(Held-out 성능 저하) 테스트해 본 결과입니다.
- 기존 (Full Finetuning): 성능 89% 폭락 (기존 지식 완전 붕괴)
- 희소 메모리 (Sparse Memory): 성능 저하 단 11% (기존 지식 거의 완벽 보존)

💡 마치며: 시스템 최적화의 본질은 '분리(Isolation)'
"특정 팩트를 기억하는 능력"과 "일반적인 논리 구조를 담당하는 능력"을 수학적으로 완벽히 고립(Isolation)시키는 것이 이 파괴적 논문의 본질입니다. 

실시간으로 매일 쏟아지는 사내 데이터를 지식화해야 하는 기업용 B2B 에이전트 시스템에서 "내일도 과연 이놈이 똑똑할까?" 하는 불안감을 완전히 지워버린 훌륭한 백신이 탄생했습니다.
